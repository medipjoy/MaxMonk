01 Product Overview
Core Philosophy

ClearDay is built around the Eisenhower Matrix as the primary mental model. Every task (called a Move) lives on a free-flowing XY canvas where X = Urgency and Y = Importance. Bubble size reflects time commitment. Everything secondary lives behind icons — the canvas is sacred.

🎯
Primary User
Target
Finance professionals managing both personal and professional goals. Individual use, but shareable read-only snapshots with teammates.
📦
Deployment
Runtime
Ships as a single React file. Runs natively in Claude artifact sandbox. Adaptable to VS Code with localStorage fallback for window.storage.

When running outside Claude, replace window.storage.get/set with localStorage.getItem/setItem. The AI calls require a proxy or CORS-enabled endpoint since browsers block direct Anthropic API calls without an API key header. Inject the key server-side or via a Vite env variable.

04 Bubble Matrix Canvas
Feature Priority

P1 · Core   This is the entire app. Everything else supports it.

Axis System
Full-viewport canvas. Flex grows to fill remaining height after MIT strip and topbar.
SVG overlay renders two crossing gradient lines: horizontal = URGENCY axis, vertical = IMPORTANCE axis.
Arrowheads on right (urgency) and top (importance) ends of axes.
Axis labels: URGENCY bottom-right of horizontal, IMPORTANCE top-right of vertical. Font-size 7px, opacity 10%.
Quadrant watermarks: DO NOW (Q1, top-right), SCHEDULE (Q2, top-left), DELEGATE (Q3, bottom-right), ELIMINATE (Q4, bottom-left). 9px, ~8% opacity, colored per quadrant.
SVG is position:absolute, inset:0, pointerEvents:none. Canvas div has cursor:crosshair.

Q1 — Do Now

Urgent · Important

  • Top-right quadrant (cx>0.5, cy<0.5)
  • Bubble color: #FF5C5C
  • Highest z-index among active

Q2 — Schedule

Important · Not Urgent

  • Top-left quadrant (cx<0.5, cy<0.5)
  • Bubble color: #2ECC8F
  • Goal: most moves should live here

Q3 — Delegate

Urgent · Not Important

  • Bottom-right (cx>0.5, cy>0.5)
  • Bubble color: #5B9BFF
  • On hold bubbles dim to 38% opacity

Q4 — Eliminate

Neither

  • Bottom-left (cx<0.5, cy>0.5)
  • Bubble color: #6B6B82
  • Lowest visual weight
05 Move Bubbles
🫧
Bubble Rendering
Visual
Position: absolute, computed from cx * canvasWidth and cy * canvasHeight.
Size (radius r) determined by time field: quick=20, short=29, medium=38, deep=50.
Background: radial gradient from color22 to color08 (hex alpha suffix).
Border: 1.5px solid color40. On hold: color20. Dragging: color66.
Text: first 1–3 words of move text (1 for quick, 2 for short, 3 for medium/deep) + "…" if truncated.
Domain icon (🏠 personal / 💼 professional) at bottom, 7px, 40% opacity. Tap to toggle domain.
On hold: pause icon top-right at 50% opacity.
Entry animation: pop keyframe (scale .35 → 1.07 → 1, cubic-bezier(.34,1.56,.64,1)).
⏱️
Time → Size Mapping
Logic
quick ⚡ — ≤30 min — radius 20px
short ◔ — ~1 hour — radius 29px
medium ◑ — ~2 hours — radius 38px
deep ● — 3h+ — radius 50px
User selects via icon-only button row in Add Move panel.
Default: short.
Add Move Panel — Full Spec
Triggered by: tapping canvas background, or tapping + icon in any quadrant (legacy, optional).
Tapping canvas pre-initializes sliders based on tap position: converts (tapCx, tapCy) → urgency/importance via slidersFromPos().
Textarea: full-width, auto-focus, min-height 58px. Enter (no shift) submits.
Urgency slider: range 5–95. Gradient fill: Q1 red to transparent. Label shows Very Low / Low / High / Very High.
Importance slider: range 5–95. Gradient fill: Q2 green to transparent.
Mini Matrix Preview (44×44px, 2×2 grid): live updates as sliders change. Active quadrant shows colored dot.
Quadrant tag: auto-derived from sliders. urg>50 + imp>50 = Q1, urg>50 + imp<50 = Q3, etc. Shown as colored badge.
Time selector: 4 icon buttons (⚡◔◑●). Selected state: purple border + purple text.
Domain toggle: Professional / Personal. Full-width split buttons.
Submit button label: "Place in {QS[q].l}". Disabled until text is non-empty.
On submit: calls addMove(text, q, domain, time, cx, cy) where cx/cy derived from sliders via posFromSliders(urg, imp).
06 Drag & Drop
Implementation Approach

No drag library. Global mousemove/mouseup and touchmove/touchend listeners on window, attached once via useEffect. Drag state held in a useRef (not state) to avoid stale closures in the listeners.

👆
Drag Initiation
Interaction
onMouseDown / onTouchStart on each bubble div.
Stores: id, startX, startY, origCx, origCy, rect, moved=false, clickCb.
clickCb = function to open detail panel for that move.
Bubble cursor: grab. Active: grabbing.
🔄
Live Drag Behavior
Interaction
Threshold: moved=true when delta > 0.008 of canvas dimension.
Live position stored in dragOverRef (ref) AND dragOver (state) for render.
Bubble reads dragOverride prop for live position during drag.
Bubble color updates live as it crosses quadrant boundary.
Dragging bubble: scale 1.08, box-shadow glow, z-index 30, no animation.
Bounds: cx clamped 0.03–0.97, cy clamped 0.03–0.97.
Drop / Release
If moved === false: treat as click, call clickCb() to open detail panel.
If moved === true: compute final quadrant from qFromPos(cx, cy). Update move with new cx, cy, quadrant.
Persist to storage immediately on drop via setMoves_(prev => ...) inside the listener (uses functional update to avoid stale state).
Clear dragRef.current, dragOvRef.current, and setDragOver(null) on mouse/touch up.
07 MIT Strip
Positioned between topbar and canvas. Fixed height ~36px. Flex-shrink: 0.
Background: subtle purple gradient. Border: 1px solid rgba(167,139,250,.13).
Star icon (filled purple if MIT set, outline if not) + "MIT" label in muted text.
Click anywhere on strip to enter edit mode (inline input, auto-focus).
Enter key saves. Escape cancels. onBlur saves.
MIT is stored per-day: key cd-mit-YYYY-MM-DD. Resets each new day automatically.
Any Move can be promoted to MIT from its detail panel ("Set MIT" button).
MIT is included in End-of-Day AI reflection and Share snapshots.

09 On Hold
Accessible via pause ⏸ icon in top bar. Count badge shows number of paused moves.
On Hold moves remain on canvas at 38% opacity with a small pause icon overlay.
On Hold moves can still be dragged. They remain at their position.
On Hold panel shows list with: text, quadrant label, Resume (▶) and Vault (archive) actions.
Resume restores status to "active", full opacity.
Move detail panel also offers Pause/Resume as an action button.
10 Vault
Soft Delete Design

Moves are never immediately deleted. They go to the Vault where they live for 60 days. After 60 days they enter an "expired" state and are eligible for permanent deletion. The user must manually confirm permanent delete.

Archiving a Move
Available from: Move detail panel ("Vault" button) or On Hold panel (archive icon).
Move is removed from moves array, added to vault array with archivedAt: Date.now().
Toast: "Vaulted · 60 days to recover".
If detail panel is open for that move, it closes automatically.
Vault Panel
Dedicated 🗄 icon in top bar with count badge.
Active items: show text, quadrant, days remaining. Color turns red when <10 days left.
"reignite" button (↩ icon + text): restores move to matrix at original position. Toast: "Restored to matrix!".
Trash icon: permanent delete (no confirm dialog — use toast undo pattern in future).
Expired section (≥60 days): strikethrough text, red tint, permanent delete only.
11 Move Detail Panel
Triggered by: click on bubble (when not dragged).
Shows: full move text, quadrant badge, domain icon+label, time icon+label, on-hold badge if held.
Urgency and importance shown as thin horizontal progress bars derived from slidersFromPos(cx, cy).
Reassign section: 3 buttons for other quadrants. On select: updates quadrant + randomizes position within new quadrant bounds. Closes panel.
Action grid (2-col): Complete ✓ · Set MIT ★ · Pause/Resume ⏸/▶ · Vault 🗄
"Complete" hidden if already done. "Pause" shows "Resume" if on hold.
Set MIT: writes move text to today's MIT store. Closes panel.
12 AI Features

AI Error Handling

All AI calls are wrapped in try/catch returning empty string on failure. Spark categorization has a hardcoded fallback object if JSON.parse fails. Never throw to the user — show empty state or default values silently.

Generating a Code
Generate button: creates random 6-char uppercase alphanumeric code via Math.random().toString(36).slice(2,8).toUpperCase().
Snapshot stored in shared storage: key cd-sh-{CODE}, value: {moves, mit, name, ts}.
Only non-done moves included in snapshot.
Code persisted to cfg (cd-cfg). Displayed large monospace. Copy button.
Regenerate = new code, new snapshot. Old code becomes inaccessible (no delete needed — just abandoned).
Viewing a Code
Input auto-uppercases. Max 6 chars.
Eye button fetches cd-sh-{CODE} from shared storage.
Snapshot shown inline: owner name, MIT, moves grouped by quadrant.
Read-only. No actions. No drag. No edit.
Not found: toast error "Code not found".

Implementation Notes for VS Code
1. Replace window.storage with a localStorage wrapper using the same async API shape.
2. AI calls require an API key — use a Vite proxy (vite.config.js server.proxy) or inject via import.meta.env.VITE_ANTHROPIC_KEY and send as x-api-key header.
3. Google Fonts import works via @import url() inside the injected <style> tag — no config needed.
4. ResizeObserver on the canvas ref works natively — no polyfill needed for modern browsers.
5. Touch events work as-is on mobile. Test drag on iOS Safari — e.cancelable check before preventDefault() is already in place.
6. Single file: all components, styles, helpers, and the default export in one App.jsx. No file splitting required.
